- Markdown Syntax and Spacing

        
    **bold**
    </center>
    
    ---
    $$
            \frac{n!}{k!(n-k)!} = \binom{n}{k}
    $$
        

    

    
    

        
Explanation:

        
Proper spacing between Markdown syntax and other HTML or Markdown elements is crucial for correct rendering. Markdown relies on specific patterns to recognize and parse its syntax.

        
When Markdown syntax is directly adjacent to HTML or other Markdown syntax without a line break, it may not be parsed correctly. This can result in unexpected behavior or rendering issues.

        
To ensure correct rendering:

        
    
            
  • Always leave at least one line space between Markdown syntax and any adjacent HTML or Markdown elements.
    • 
            
  • This ensures that Markdown parsers can accurately identify and parse the Markdown syntax, leading to consistent and expected rendering.
    • 
        

        
Example:

        
    **bold**
    </center>
    << This wont render correctly, Because there is no line break b/w '---' & '$$' >>

    ---
    $$
            \frac{n!}{k!(n-k)!} = \binom{n}{k}
    $$
        

        
In the provided example, the Markdown syntax for bold text and the closing HTML tag for center alignment are directly adjacent without a line break, leading to potential parsing issues. Adding a line break between them ensures correct rendering.

        
            **bold**
            </center>
            << This will render correctly, Because there is line break b/w '---' & '$$' >>
            ---
            << Line Break Here >>
            $$
                    \frac{n!}{k!(n-k)!} = \binom{n}{k}
            $$
        

        

            
Proper spacing between Markdown syntax and other HTML or Markdown elements is crucial for correct rendering. 
 Markdown relies on specific patterns to recognize and parse its syntax.

            
When Markdown syntax is directly adjacent to HTML or other Markdown syntax without a line break, it may not be parsed correctly. This can result in unexpected behavior or rendering issues.

            
Adding a line break between them ensures correct rendering. This principle applies to every other Markdown syntax as well.

            
Always keep line breaks between each Markdown syntax.

        

    

    

    

    








    

        
- Use Markdown Syntax within HTML Tags

        
        

    

    
    

        
Explanation:

        
You can use Markdown syntax within HTML tags to combine the flexibility of Markdown with the power of HTML. This feature allows you to create rich content with ease.

        
To use Markdown syntax inside HTML tags, follow these steps:

        
    
            
  1. Start with an HTML tag, such as <center>.
    2. 
            
  2. Leave the first line empty after the HTML tag.
    3. 
            
  3. Write your Markdown content, such as headings, lists, or emphasis, on subsequent lines.
    4. 
            
  4. End with the corresponding closing HTML tag, such as </center>.
    5. 
        

        
Here's an example:

        
    <center>
        comment this line out: Leave this line empty, Start writing from next line...
        - Item 1
        - Item 2
        **bold text**
    </center>
        

        
This will result in the Markdown content being rendered centered within the HTML element.

Tables

        
    | Header 1 | Header 2 |
    | -------- | -------- |
    | Cell 1   | Cell 2   |
    | Cell 3   | Cell 4   |
        

    

    
    

        
Explanation:

        
Markdown supports the creation of tables using a simple syntax. Tables consist of rows and columns, with cells separated by pipe (|) characters. The first row defines the table headers, and subsequent rows define the content of each cell.

        
To create a table:

        
    
            
  1. Start a new line.
    2. 
            
  2. Define the headers in the first row, separating each header with a pipe character (|).
    3. 
            
  3. Insert a second row with hyphens (-) to indicate the column alignment.
    4. 
            
  4. Add rows of content below, with cells separated by pipe characters (|).
    5. 
        

        
Example:

        
    | Header 1 | Header 2 |
    | -------- | -------- |
    | Cell 1   | Cell 2   |
    | Cell 3   | Cell 4   |
        

        
This will create a table with two columns and two rows.
